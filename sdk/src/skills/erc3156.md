# ERC-3156

Standard interface for flash loans (EIP-3156). Provides flashLoan, flashFee, and maxFlashLoan functions for any conforming lender.

## Category
infrastructure | Chains: (standard interface, any chain)

## Key Operations
- **flashFee**: Query the fee for a flash loan of a specific token and amount
- **maxFlashLoan**: Query the maximum flash loan amount for a token
- **flashLoan**: Execute a flash loan (borrow and repay in same transaction)

## SauceScript Examples
```typescript
// Query flash loan fee
import { FlashLenderABI as IFlashLender } from "./abis";
function main(lenderAddress: Address, token: Address, amount: Uint256): Uint256 {
  const lender = IFlashLender.at(lenderAddress);
  return lender.flashFee(token, amount);
}

// Query max flash loan amount
import { FlashLenderABI as IFlashLender } from "./abis";
function main(lenderAddress: Address, token: Address): Uint256 {
  const lender = IFlashLender.at(lenderAddress);
  return lender.maxFlashLoan(token);
}
```

## ABI Methods
### FlashLenderABI
- `flashLoan(address,address,uint256,bytes)` - Execute flash loan. Params: receiver (contract that receives tokens and implements IERC3156FlashBorrower), token (ERC-20 to borrow), amount (borrow amount), data (arbitrary data passed to receiver callback). Returns bool success
- `flashFee(address,uint256)` - Query fee for flash loan. Params: token, amount. Returns fee in token units
- `maxFlashLoan(address)` - Query max borrowable amount. Params: token. Returns max amount (0 if token not supported)

## Notes
- Standard interface (EIP-3156), not a specific deployment - no fixed addresses
- Aave V3, Maker (sDAI), Uniswap V3, and Balancer all implement ERC-3156
- Receiver contract must implement `onFlashLoan(address,address,uint256,uint256,bytes)` callback
- Must approve lender to pull back amount + fee before callback returns
- Flash loan must be fully repaid within the same transaction (atomic)
- Fee varies by implementation: Aave charges 0.05-0.09%, Balancer charges 0%, Maker varies
